230: /*MC
231:   TSRegisterDynamic - Adds a creation method to the TS package.

233:   Synopsis:
234:   PetscErrorCode TSRegisterDynamic(const char *name, const char *path, const char *func_name, PetscErrorCode (*create_func)(TS))

236:   Not Collective

238:   Input Parameters:
239: + name        - The name of a new user-defined creation routine

241: . func_name   - The name of the creation routine
242: - create_func - The creation routine itself

244:   Notes:
245:   TSRegisterDynamic() may be called multiple times to add several user-defined tses.

255: .vb
256:     TS ts;

260:   or at runtime via the option
261: .vb
262:     -ts_type my_ts
263: .ve

265:   Notes: $PETSC_ARCH occuring in pathname will be replaced with appropriate values.
266:         If your function is not being put into a shared library then use TSRegister() instead

268:   Level: advanced

270: .keywords: TS, register
271: .seealso: TSRegisterAll(), TSRegisterDestroy()
