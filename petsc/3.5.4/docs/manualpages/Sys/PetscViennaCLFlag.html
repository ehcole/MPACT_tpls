PetscViennaCLFlag

 indicates which memory (CPU, GPU, or none contains valid vector

Synopsis

typedef enum {PETSC_VIENNACL_UNALLOCATED,PETSC_VIENNACL_GPU,PETSC_VIENNACL_CPU,PETSC_VIENNACL_BOTH} PetscViennaCLFlag;
PETSC_VIENNACL_UNALLOCATED - no memory contains valid matrix entries; NEVER used for vectors PETSC_VIENNACL_GPU - GPU has valid vector/matrix entries PETSC_VIENNACL_CPU - CPU has valid vector/matrix entries PETSC_VIENNACL_BOTH - Both GPU and CPU have valid vector/matrix entries and they match
